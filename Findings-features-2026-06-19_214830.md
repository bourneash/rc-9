# Feature & Improvement Findings — Remote Command (rc-9.com)

Scope: Whole-product review against the stated income plan (`INCOME_PLAN.md`) and best-in-class browser/portal artillery games (Scorched Earth, Worms, ShellShock Live, Tank Trouble). Covers monetization, multiplayer, retention/onboarding, analytics, ops/observability, a11y, mobile/PWA, build/CI, and docs. Game-feature ideas are intentionally generous; each is framed as a concrete, buildable item.

## 🚀 Major — missing capabilities
- [ ] **[ads/revenue]** Wire up the rewarded + preroll ad system that already exists but is never called. `ads.js` defines `showPreroll` and `showRewarded`, but no game code invokes them and `GAME_ID` is still `__GAME_ID_REPLACE_ME__`. Implement the three integration points from the plan — **revive** ("watch ad to revive with 25 HP"), **weapon unlock** ("watch ad to unlock Nuke this round"), **fuel refill** — set a real GameDistribution game id at build time, and add the SDK domain to CSP. This is the income plan's single biggest revenue lever and is currently 100% dead. (See bugs file for the CSP/placeholder blockers.)
- [ ] **[multiplayer]** Online multiplayer. The game is hot-seat/AI only (`CLAUDE.md` "Network Play: single-player/hotseat only"). Real-time or async online play (Cloudflare Durable Objects + WebSockets fit the existing CF stack perfectly) is the #1 table-stakes feature competitors have and the strongest retention/virality driver for an artillery game.
- [ ] **[game]** Save / resume a match. Only settings persist to localStorage; an in-progress battle is lost on refresh or accidental navigation. A `snapshot.js`/`snapshot-validator.js` serialization layer already exists — surface it as "Resume last game" + autosave-on-turn so long hot-seat matches survive a reload.
- [ ] **[retention]** Player progression + persistent stats: wins/losses, accuracy, favorite weapon, win streaks, unlockable cosmetics, and a local (then global) leaderboard. Portal traffic is high-volume and low-loyalty — progression is what converts a one-time play into a returning visitor and lifts ad impressions per user.
- [ ] **[onboarding]** First-run interactive tutorial. Artillery aim/power/wind mechanics are non-obvious to new players arriving cold from a portal; a 60-second guided "fire your first shot" flow measurably reduces bounce on game portals. There is a `help.html` page but no in-game onboarding.

## ⭐ High-impact improvements
- [ ] **[analytics]** Make analytics actually work end-to-end: fix the CSP so GA4 can load (or switch fully to the cookieless Cloudflare Web Analytics already embedded), adopt **GA4 Consent Mode v2**, and add custom events for the funnel that matters — game_start, match_complete, weapon_used, rewarded_ad_shown/completed, revive_taken. Right now you have zero behavioral data to optimize the income plan against.
- [ ] **[observability]** Production error telemetry. `errors.js` collects errors only in memory and never reports them, and there's no `window.onerror`/`unhandledrejection` hook. Wire a lightweight sink (Sentry, or a Worker `/api/log` endpoint writing to a CF queue/Analytics Engine) so crashes from real portal traffic are visible. A 12k-line client game shipping to 6 portals is currently flying blind.
- [ ] **[a11y]** Accessibility pass. Canvas gameplay has no screen-reader narration of game state, and the heavy title-screen/particle animations ignore `prefers-reduced-motion`. Add reduced-motion guards, ensure all menu controls are keyboard-reachable with visible focus, and add `aria-live` turn/score announcements. Improves reach and portal acceptance.
- [ ] **[mobile/pwa]** Add a web app manifest + service worker so the game is installable and works offline (Phase 3 PWA goal, and a Play-Store-via-Bubblewrap prerequisite). None exists today (`manifest.webmanifest`/`sw.js` absent), so there's no "Add to Home Screen", no offline cache, and no AdMob path.
- [ ] **[security]** Add CSP violation reporting (`report-to`/`report-uri`) to `_headers`. A reporting endpoint would have immediately surfaced the inline-script breakage in the bugs file; going forward it catches third-party/CSP regressions before they silently break consent or ads in production.
- [ ] **[content/seo]** Build the `/games` retro hub (income plan Phase 2.1) to keep 100% of ad revenue on-domain and compound SEO across multiple titles. Even a single second game on the domain materially raises the traffic floor and session depth.
- [ ] **[balance]** Difficulty/progression beyond the 3 flat AI levels: a single-player campaign or escalating-AI ladder gives solo players (the majority of portal traffic) a reason to keep playing past one match.

- [ ] **[a11y/ux]** Respect `prefers-reduced-motion` on the animated boot/title sequence and the canvas particle load — currently always-on, which is jarring and a vestibular-accessibility issue.
