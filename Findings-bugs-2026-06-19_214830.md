## 🔴 Critical
- [ ] **[security/compliance]** The CSP `script-src` has no `'unsafe-inline'`, nonce, or hash, but the page depends on **2 inline `<script>` blocks and 5 inline event handlers** (`index.html:12-28` GA4 consent loader, `index.html:740-751` cookie-banner controller, `onclick="rc9Accept()"`/`rc9Decline()`/`localStorage.removeItem(...)` and `onmouseover`/`onmouseout` at `index.html:727,734-738`). Header at `public/_headers:10`: `script-src 'self' https://static.cloudflareinsights.com`. If the CSP is enforced in production (Cloudflare Workers static assets honor `_headers`), **every inline script is blocked**: the cookie banner never un-hides (it ships `opacity:0;pointer-events:none` and is only revealed by the blocked script), and `window.rc9Accept`/`rc9Decline` are never defined — so the entire cookie-consent mechanism is dead on every visit. The site advertises analytics cookies with no working consent UI. **VERIFY LIVE** with `curl -I https://rc-9.com` + browser console; if confirmed, add a nonce/hash or move these scripts to bundled modules.

## 🟠 High
- [ ] **[analytics]** Consent-gated GA4 (`G-R2H86NCJ2F`) can never load even if its inline loader ran: CSP `script-src` omits `https://www.googletagmanager.com` and `connect-src` (`public/_headers:10`) omits `https://www.google-analytics.com`. The gtag library and all measurement hits are blocked → **zero analytics is being collected**, despite the income plan depending on traffic data. — `index.html:12-28`, `public/_headers:10`
- [ ] **[physics]** Per-weapon `gravityFactor` is silently collapsed to 1 — `projectile.js:259` (`this.gravityFactor = stats.gravityFactor === 0 ? 0 : 1`). Every fractional value (homing 0.25, torpedo 0.2, depth_charge 1.2, underwater_mine 0.1, homing_torpedo 0.15, navy_seal 0.3) is discarded, so those weapons fall/sink at full gravity. — Fix: `stats.gravityFactor != null ? stats.gravityFactor : 1`. Breaks flight/sink balance on every shot of those types.
- [ ] **[game/race]** Stale AI fire after fast "New Game" — `game-ai.js:64-93` (and the first-AI-turn kick at `game.js:1242-1247`). The `setTimeout(thinkTime)` + recursive `tryFire` chain only guards on `gameOver`/`paused`, never on a turn token or `aiTank ∈ game.tanks`. Restarting during the ~1.5s think window lets the old closure fire a phantom projectile into the fresh game and clobber `aiTurnInProgress`/`isAnimating`/`turnEnding`. Ghost shots + corrupted turn state.
- [ ] **[game]** `homing_torpedo` homes onto its own owner and teammates — `projectile.js:287-301` (targets from `game.js:1559`). Steering loop only skips `t.health <= 0`; it never excludes `this.owner` or friendlies (the air `homing` path correctly uses `getNearestEnemyTank`). A fired torpedo can curve back and detonate on the shooter/ally.

## 🟡 Medium
- [ ] **[ci]** The CI gate `npm run security:audit` = `npm audit --audit-level=high` includes devDependencies; there are currently **5 high advisories** (vite, ws, launch-editor — all dev-only, not shipped). The "Dependency audit (high+)" step (`.github/workflows/security-and-build.yml:32-33`) fails (or is failing) the pipeline on vulnerabilities that never reach users. Use `--omit=dev` for the gate (the `security:audit:prod` script already exists). — `site/package.json:13-14`
- [ ] **[infra]** Missing `/404` page. The Worker redirects `/admin/*`, `/api/*`, and unknown `/go/<id>` to `/404` (`worker/index.js:21,25`) and `wrangler.jsonc` sets `not_found_handling: "404-page"`, but **no `404.html` exists** in `public/` or `dist/` → those redirects land on a page that itself 404s. — Add `public/404.html`.
- [ ] **[ads/revenue]** Ad monetization is entirely non-functional: `ads.js` auto-loads the GameDistribution SDK on production, but (a) the SDK domain `html5.api.gamedistribution.com` is **not in CSP `script-src`** so the injection is blocked (`ads.js:42-45`), (b) `GAME_ID` is still the placeholder `'__GAME_ID_REPLACE_ME__'` (`ads.js:5`), and (c) **nothing calls `showPreroll`/`showRewarded`** (grep across `js/` finds only the definitions). No ads can render. — `ads.js`, `js/init.js:16`
- [ ] **[compliance]** `ads.js.shouldEnable()` returns true for all production loads and injects the third-party GameDistribution ad SDK with **no consent gating** (`ads.js:16-25,41-46`), contradicting the privacy policy's "We have no other third-party data relationships" (`privacy.html`) and the cookie banner's analytics-only scope. Currently masked only because CSP blocks the domain — re-enabling ads without fixing this is a live GDPR/ePrivacy exposure.
- [ ] **[input]** `isFormTarget` guards on `getElementById('debug-modal-overlay')`, but the real element id is `debug-modal` (`index.html:316`); the overlay id exists nowhere — so game keys (fire/move) are **not** suppressed while the debug modal is open; interacting with debug sliders can trigger gameplay actions. — `main.js:3571`
- [ ] **[input]** The Escape branch in `onKeyDown` opens `game-controls-modal`, which does not exist in any HTML/JS, so the intended "Escape opens Game Options" feature is dead and Escape silently swallows the key (`e.preventDefault(); return;` runs unconditionally). — `main.js:3636-3646`
- [ ] **[stats]** `turnCount` never resets between games — initialized at `game.js:119`, incremented at `5590`, but absent from `reset()` (`7370-7505`) and `startNewGameWithConfig()` (`674-947`). The end-of-game "ROUND COMPLETE · N TURNS" report (`er-rounds`) accumulates turns across every game played without a page reload.
- [ ] **[stats]** `turnCount++` runs *before* `endTurn()`'s early-return guards — increment at `game.js:5590`, guards at `5592-5598`. When `endTurn` bails (animation/projectiles still active, or every shot in realtime mode) the counter still increments, further inflating turn stats.
- [ ] **[game]** Air-Nukes bomber can silently drop fewer (or zero) bombs than `bombCount` — `game.js:2131-2161`. Only one drop position is tested per frame and `_lastX` is unconditionally advanced; a single large-`dt` frame (fast plane, 46px spacing, or a long frame after a tab resume) skips positions that then never re-trigger. Root cause is the **uncapped `deltaTime`** in the game loop (`game.js:1252` → `update` `1265`, no clamp).
- [ ] **[game]** `addLog()` dereferences `#log-messages` with no null guard (`insertBefore`) — `game.js:7359-7363`. Called on many common paths (turn end, deaths, `reset()` at 7504, hazard/mine kills at 7770/8112); if the log container is ever absent, every call throws `TypeError` and aborts the calling update path mid-frame.
- [ ] **[ai]** Realtime AI aim uses the wrong tank's skill after the AI repositions — `updateRealtimeAI` restores `currentTankIndex` to the human synchronously (`game.js:5575-5578`) but `calculateAIShot` (deferred via `executeAIMove`'s `setTimeout`) re-reads `game.tanks[game.currentTankIndex]` for `aiSkill` (`game-ai.js:280`). dx/dy/distance are passed correctly via closure, so only the aim-error model is wrong.

## ⚪ Low
- [ ] **[game]** `checkGameOver()` is not idempotent (no `if (game.gameOver) return;`) — `game-victory.js:6-25`. Multiple post-end call sites (`game.js:5461/5147/5642`) can each schedule another 2s `showGameOver`/`openGameOverModal`, so the game-over modal can open more than once within a single game.
- [ ] **[robustness/latent]** A `NaN` projectile would never cull or collide (`<`/`>=` comparisons all false for NaN at `game.js:1566-1571,1622,1701`), so `projectiles.length` never reaches 0 and the turn never ends — a soft freeze. No NaN source found on the common path (divisions are `|| 1`-guarded); latent only. Add a `Number.isFinite` cull as defense.
- [ ] **[ui]** `main.js` reads canvas + `getContext('2d')` with no null check at module load (`main.js:160-161`); if `#game-canvas` were ever absent the module throws before the nice fallback UI (`main.js:255-283`) can render. Element exists today, so latent.
- [ ] **[validation]** Tank `color` from `se.slots.v1` localStorage / swatches is passed to the engine without `validateColor()` (which exists in `validation.js:74` but is unused) — `main.js:2065`. Not XSS (used only as canvas `fillStyle`), but a hand-edited/corrupted value yields silently broken tank rendering with no fallback.
- [ ] **[infra]** `worker/index.js:24-25` blocks `/admin/*` and `/api/*` with a 302 redirect to `/404` rather than returning a 404/403 status — a soft, cache-unfriendly block (and currently lands on a missing page; see Medium). Low impact since there is no real admin/API surface.
- [ ] **[errors]** `errorLogger.warn()` only logs in development (`errors.js:49-52`) and `errorLogger` is never wired to `window.onerror`/`unhandledrejection` — uncaught production errors are not captured anywhere. Stored errors live only in memory (`errors.js:8`) and are never reported off-device.
- [ ] **[cleanup]** Dead/stale code: `suppressOptionsUntil` is written (`main.js:216,1317`) but never read; smoke spec still probes `window.game`/`window.mainGame` aliases that are no longer set (per `CLAUDE.md`); `[BOMBER DEBUG]` `console.log` at `game.js:2138-2140` (stripped from prod by terser `drop_console`, but noise in dev).
- [ ] **[ci]** `gitleaks` and `semgrep` steps are both `continue-on-error: true` (`.github/workflows/security-and-build.yml:59,77`) — secret-scan and SAST never fail the build, so any finding they surface can still merge to `main`. Treat as informational-only or make blocking.
- [ ] **[game]** Spawn fallback ignores the slope check and the canyon fallback returns unsorted spawn points (`game.js:7132-7180`) — cosmetic: tanks may spawn on steep slopes or out of left-to-right order. Toxic-gas hazard radius also grows unbounded within its lifetime (`game.js:7716`, `h.radius += 0.01*(dt/16)` with no clamp) — minor balance drift on long-lived clouds.
